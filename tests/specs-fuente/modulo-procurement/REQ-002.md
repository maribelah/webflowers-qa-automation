# REQ-002 — Ingreso al módulo New Purchase Order

**Módulo:** Procurement  
**Requerimiento:** REQ-002  
**Analista QA:** QA Automation  
**Estado:** Cerrado

## Descripción
Verificar que un usuario autenticado pueda navegar exitosamente al módulo New Purchase Order 
a través del menú principal (Procurement → Products → New PO) y que el formulario del módulo 
se cargue correctamente.

## Flujo principal
1. Navegar a la URL del ambiente activo
2. Ingresar credenciales válidas desde JSON de datos
3. Hacer clic en el botón de ingresar
4. Validar que el Dashboard carga correctamente
5. Navegar a Procurement → Products → New PO usando `navigateToNewPurchaseOrder()`
6. Validar que el formulario de New PO esté visible en `frameCenter#divForm`

**Resultado esperado:** El formulario de New Purchase Order está visible y listo para ingreso de datos.

## Casos de prueba

### ✅ Caso exitoso
- **Nombre:** Login exitoso + navegación a New PO
- **Datos:** Usuario y contraseña válidos desde `REQ-002-data.json.exitoso`
- **Validación:** Formulario `#divForm` visible en `frameCenter`

### ❌ Caso fallido
- **Nombre:** Login con contraseña incorrecta
- **Datos:** Usuario válido, contraseña incorrecta desde `REQ-002-data.json.fallido`
- **Validación:** Mensaje de error visible en pantalla de login

### 🔲 Caso borde
- **Nombre:** Login con campos vacíos
- **Datos:** Usuario y contraseña vacíos desde `REQ-002-data.json.borde`
- **Validación:** Validación de formulario — campo requerido
